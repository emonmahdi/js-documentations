## ‚úÖ **1) Class vs Prototype ‚Äî Interview Answer**

**Answer:**
JavaScript internally prototype-based, ‡¶Ö‡¶∞‡ßç‡¶•‡¶æ‡ßé ‡¶™‡ßç‡¶∞‡¶§‡¶ø‡¶ü‡¶ø object ‡¶è‡¶∞ ‡¶è‡¶ï‡¶ü‡¶ø hidden property ‡¶•‡¶æ‡¶ï‡ßá ‚Äî `[[Prototype]]`.
Class ‡¶π‡¶≤‡ßã just **syntactic sugar** ‡¶Ø‡¶æ prototype ‡¶ï‡ßá ‡¶∏‡¶π‡¶ú ‡¶ï‡¶∞‡ßá‡•§

‚úî Prototype ‚Üí ‡¶™‡ßÅ‡¶∞‡¶®‡ßã system (ES5)
‚úî Class ‚Üí modern readable syntax (ES6)

**Example:**

```js

  this.name = name;
}
Person.prototype.say = function () {
  console.log(this.name);
};
```

vs

```js

  constructor(name) {
    this.name = name;
  }
  say() {
    console.log(this.name);
  }
}
```
