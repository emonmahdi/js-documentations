ЁЯОп Interview Level Definition

> Call Stack is a data structure that keeps track of function execution in JavaScript. It follows the Last In First Out (LIFO) principle. When a function is called, its execution context is pushed onto the stack, and when it finishes execution, it is popped off.

--------------------------------------

# ЁЯФе Critical Interview Q&A

---

### тЭУ 1. What is Call Stack?

Answer:
Call Stack is a LIFO data structure used by JavaScript engine to manage execution contexts of function calls.

---

### тЭУ 2. How does Call Stack work?

Answer:
When a function is invoked, a new execution context is created and pushed to the stack. When the function completes, it is removed (popped) from the stack.

---

### тЭУ 3. Why does Stack Overflow happen?

Answer:
Stack overflow happens when too many function calls are pushed into the call stack without being removed, usually due to infinite recursion.

---

### тЭУ 4. Is Call Stack part of JS engine?

Yes. It is part of the execution mechanism of the JS engine.

---

### тЭУ 5. Difference between Call Stack and Heap?

| Call Stack               | Heap              |
| ------------------------ | ----------------- |
| Stores execution context | Stores objects    |
| LIFO structure           | Memory allocation |
| Small & limited          | Larger memory     |

---

### тЭУ 6. How does Call Stack handle async code?

It does not directly handle async code. Async tasks are handled by Web APIs and Event Loop, which push callbacks to the Call Stack when it becomes empty.
