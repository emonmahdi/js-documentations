1. What is JavaScript?

JavaScript is a high-level, interpreted programming language primarily used to add interactivity and dynamic behavior to websites. It runs in the browser and allows developers to:

  • Manipulate the DOM (Document Object Model)
  • Handle user events (clicks, input, etc.)
  • Make asynchronous requests using AJAX or fetch
  • Build interactive UI, games, SPAs (Single Page Applications)

JavaScript is also used in server-side development (e.g., with Node.js), making it a full-stack language. 

// JavaScript example: Change text on button click
                    document.getElementById("myBtn").addEventListener("click", function() {
                        document.getElementById("demo").innerText = "Hello JavaScript!";
                    });
2. What are the data types in JavaScript?

JavaScript has six primitive data types:

  • string
  • number
  • boolean
  • null
  • undefined
  • symbol (introduced in ES6)

It also has one non-primitive (complex) data type:

  • object ‚Äì used for arrays, functions, and other objects.

// Primitive types
                    let name = "Emon";       // string
                    let age = 25;            // number

                    let greet = function() { console.log("Hi"); };
3. What is the difference between null and undefined?

The value null represents the intentional absence of any object value.
The value undefined means a variable has been declared but has not yet been assigned a value.

  • null is an assigned value. It means nothing.
  • undefined means a variable has been declared but not assigned a value yet.
let a;
                console.log(a); // undefined
                
                let b = null;
                console.log(b); // null
                
                console.log(typeof a); // "undefined"
                console.log(typeof b); // "object"
4. Explain the concept of hoisting in JavaScript.

Hoisting is a JavaScript mechanism where variable and function declarations are moved to the top of their scope (global or local) during the compilation phase. This allows you to use variables and call functions before they're declared in the code.

However, only the declarations are hoisted, not the initializations. 

// Example with var
                console.log(a); // undefined
                var a = 5;
                
                // Example with let/const (temporal dead zone)
                console.log(b); // ‚ùå ReferenceError
                let b = 10;
                
                // Function hoisting
                greet(); // ‚úÖ Works due to hoisting
                function greet() {
                    console.log("Hello!");
                }

‚ö†Ô∏è Variables declared with let and const are hoisted but not initialized, leading to a ReferenceError if accessed before declaration. This period is known as the Temporal Dead Zone (TDZ).

5. What is a closure in JavaScript?

A closure is a function that retains access to variables from its outer lexical scope even after that outer function has finished executing.
Closures are useful for data encapsulation, creating private variables, and maintaining state across function calls.

Closures are created every time a function is created inside another function and the inner function accesses variables from the outer one.

function outer() {
                    let count = 0;
                
                    return function inner() {
                    count++;
                    console.log("Count:", count);
                    };
                }
                
                const counter = outer();
                counter(); // Count: 1
                counter(); // Count: 2

In this example, inner() retains access to the count variable from the outer() function even after outer() has executed.

6. What is the difference between ‚Äú==‚Äù and ‚Äú===‚Äù operators in JavaScript?

The == operator is known as the loose equality operator. It compares two values for equality after converting them to a common type (type coercion).
The === operator is the strict equality operator. It checks for equality without type conversion, meaning both the value and the data type must be the same. 

// Loose equality
                console.log(5 == '5');     // true (number vs string, type converted)
                console.log(null == undefined); // true
                
                // Strict equality
                console.log(5 === '5');    // false (different types)
                console.log(5 === 5);      // true (same value and type)

‚úÖ Use === to avoid unexpected type coercion and bugs in your code.

7. What is prototypal inheritance in JavaScript?

In JavaScript, prototypal inheritance is a method by which an object can inherit properties and methods from another object.

Every JavaScript object has an internal link to another object called its [[Prototype]]. When trying to access a property that's not found on the object itself, the JavaScript engine will look up the prototype chain until it finds it (or reaches null).

This allows for code reuse and is an alternative to classical inheritance. 

// Base object
                const animal = {
                    sound: "Generic sound",
                    speak() {
                    console.log(this.sound);
                    }
                };
                
                // New object inherits from animal
                const dog = Object.create(animal);
                dog.sound = "Bark";
                
                dog.speak(); // Output: Bark

‚úÖ Object.create() is commonly used to establish prototypal inheritance.

8. What are the different ways to define a function in JavaScript?

In JavaScript, functions can be defined in multiple ways:

  • Function Declaration
  • Function Expression
  • Arrow Function (ES6)
  • Method definition inside an object
// Function Declaration
                function greet(name) {

                // Function Expression
                const sayHello = function(name) {
                    return "Hi " + name;

                // Arrow Function
                const welcome = (name) => "Welcome " + name;
                
                // Object Method
                const person = {

                    }
                };

‚úÖ All these forms are valid and useful in different situations. Arrow functions are concise but do not have their own this context.

9. How does event delegation work in JavaScript?

Event delegation is a technique where a single event listener is added to a parent (or ancestor) element instead of attaching event listeners to individual child elements.

It works by using event bubbling‚Äîthe event starts at the target element and bubbles up to its ancestors, allowing the parent to handle the event for its children.

This is useful for improved performance and dynamically added elements. 

// HTML
                <ul id="list">
                    <li>Item 1</li>
                    <li>Item 2</li>
                    <li>Item 3</li>
                </ul>
                
                // JavaScript
                document.getElementById("list").addEventListener("click", function(e) {
                    if (e.target.tagName === "LI") {
                    alert("You clicked " + e.target.textContent);
                    }
                });

‚úÖ Event delegation is ideal when working with a large number of elements or elements that are dynamically added to the DOM.

10. What is the purpose of the ‚Äúthis‚Äù keyword in JavaScript?

The this keyword in JavaScript refers to the object that is executing the current function. Its value depends on how and where the function is called.

  • In a method, this refers to the object the method belongs to.
  • In a regular function, this refers to the global object (or undefined in strict mode).
  • In an event, this refers to the element that received the event.
  • Arrow functions don‚Äôt have their own this; they inherit it from their enclosing scope.
const user = {
                    name: "Emon",
                    greet() {
                    console.log("Hi, I'm " + this.name);
                    }
                };
                
                user.greet(); // Output: Hi, I'm Emon
                
                function sayHello() {
                    console.log(this); // In browser: Window (or undefined in strict mode)
                }
                
                sayHello();
                
                // Arrow function inherits 'this' from enclosing scope
                const arrowExample = () => {
                    console.log(this); // Same as outer context
                };

‚úÖ Understanding this is crucial for writing predictable, context-aware JavaScript functions and event handlers.

2. Difference between forEach, for, and map?
  • for: Traditional loop, flexible with break/continue.
  • forEach: Array method, no break/continue, doesn't return new array.
  • map: Returns new array, good for transformation.
const numbers = [1, 2, 3, 4];
              
              // for
              for (let i = 0; i < numbers.length; i++) {
                console.log(numbers[i]);
              }
              
              // forEach
              numbers.forEach(num => console.log(num));
              
              // map
              const doubled = numbers.map(num => num * 2);
              console.log(doubled); // [2, 4, 6, 8]

4. Difference between callback, promise, and async/await?
  • Callback: Function passed as an argument.
  • Promise: Represents future result, avoids callback hell.
  • Async/Await: Cleaner syntax to handle Promises.
// Callback
              function fetchData(callback) {
                setTimeout(() => callback("Done!"), 1000);
              }
              
              // Promise
              function fetchPromise() {
                return new Promise(resolve => {
                  setTimeout(() => resolve("Resolved!"), 1000);
                });
              }
              
              // Async/Await
              async function loadData() {
                const data = await fetchPromise();
                console.log(data); // "Resolved!"
              }
